# Инлайн-бот действий — упрощённое добавление + права админа

## Установка

```bash
pip install aiogram --break-system-packages
```

## Настройка

1. У @BotFather: `/setinline`, выбери бота, введи любой placeholder.
2. В `config.py` вставь токен и свой `user_id` в `ADMIN_IDS`:

```python
ADMIN_IDS = [
    7862735686,
]
```

(узнать свой id можно у @userinfobot)

## Запуск

```bash
python bot.py
```

## Использование

`@имя_бота обнять` в любом чате → список действий → выбор → сообщение с кнопками
✅ Принять / ❌ Отказаться → после нажатия сообщение меняется на итог.

## Добавление действия — максимально просто

```
/addaction эмодзи слово-действие
```

Пример:
```
/addaction 🤏 ущипнуть
```

Бот сам построит:
- начальный текст: "Аня хочет вас ущипнуть"
- текст согласия: "Аня ущипнул(а) Игоря" (склонение "-ть" → "-л(а)" подбирается автоматически)
- текст отказа: универсальный, не завязан на конкретное слово

Если слово не заканчивается на "-ть" (например составная фраза типа "дать пять"),
бот всё равно добавит действие с общим шаблоном, просто согласование может звучать
чуть более обобщённо — все равно рабочее.

## Права

- **Любой** может добавить действие командой `/addaction`, если добавление не закрыто админом.
- **Удалить** действие может тот, кто его добавил, либо любой админ — в любой момент.
- **Админ** может полностью закрыть добавление другими людьми:
  - `/lockadd` — закрыть (только админ сможет добавлять)
  - `/unlockadd` — снова открыть всем
- `/listactions` показывает текущий статус (открыто/закрыто).

## Ограничение

Кнопки принять/отказаться привязаны к памяти процесса — при перезапуске `bot.py` старые,
ещё не нажатые кнопки перестанут работать (покажут "действие уже неактуально").
